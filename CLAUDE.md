# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A **composite (Bash) GitHub Action** that deploys a Capacitor Live Update to Capawesome Cloud by wrapping the [`@capawesome/cli`](https://capawesome.io/docs/cloud/cli/). All logic lives in `action.yml` — there is no `src/`, no build step, and no runtime dependencies.

## Architecture

- `action.yml` is the entire implementation. Two `shell: bash` steps:
  1. **Mask secrets** — masks the token line-by-line (`mask()` helper) with workflow-command escaping (`%`→`%25`, `\r`→`%0D`); required because `::add-mask::` only masks one line per command.
  2. **Create channel** (if `channel` set) via `apps:channels:create --ignore-errors`, then upload via `apps:liveupdates:upload`.
- Auth is via the `CAPAWESOME_TOKEN` env var (no `login` step). The CLI version is pinned inline (`npx -y @capawesome/cli@<version>`).
- Optional inputs are appended to a bash `args` array only when set, using `if` blocks (not `&&`) to stay `set -euo pipefail`-safe.
- `package.json` exists **only** for `commit-and-tag-version` (release/changelog). It is not part of the action runtime.

## Conventions

- Use Conventional Commits — releases and `CHANGELOG.md` are generated by `commit-and-tag-version`. Breaking changes are documented in `BREAKING.md`.
- When changing inputs, verify CLI flag names against the pinned `@capawesome/cli` version (zli accepts both kebab-case and camelCase; kebab is canonical). Pass `${{ inputs.* }}` through step `env:` and quote `"$VAR"` — never interpolate inputs directly into `run:`.

## Commands

- `npm run release` — bump version, generate changelog, tag (run locally; needs `npm install` first).
- CI (`.github/workflows/ci.yml`) runs a live smoke test against Capawesome Cloud and requires the `CAPAWESOME_APP_ID` / `CAPAWESOME_TOKEN` repo secrets.
